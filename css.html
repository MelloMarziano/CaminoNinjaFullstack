Fundamentos: cascada, especificidad y herencia

/* Especificidad: ID > clase > tipo */
#hero { color: red; }
.hero { color: blue; }
section { color: green; }

/* Herencia: algunos props heredan (color, font) */
body { font-family: system-ui; color: #333; }
main { /* hereda color y font */ }

/* Importancia: !important (usa con moderación) */
.notice { color: orange !important; }
  • Cascada: orden de aparición + origen + especificidad + importancia.
  • Evita usar !important; prefiere selectores claros y arquitectura.
  • Resetea estilos con normalización (por ej. modern-normalize) si aplica.

Box model

.card { box-sizing: border-box; /* recomendado */ }
.box {
  width: 200px; height: 100px;
  padding: 16px; border: 2px solid #333; margin: 12px;
}
  • Usa box-sizing: border-box para calcular tamaños de forma intuitiva.

Unidades y tipografía

html { font-size: 16px; }
h1 { font-size: 2rem; /* relativo al root */ }
p { font-size: 1em; /* relativo al parent */ }
.box { width: 50vw; height: 40vh; }
  • Usa rem para tamaños consistentes; em para escalado contextual.
  • vw/vh dependen del viewport; cuidado con móviles y barras.

Colores y fondos

:root { --brand: #0d6efd; }
.btn-brand { background: var(--brand); color: #fff; }
.gradient { background: linear-gradient(135deg, #0d6efd, #6610f2); }

Flexbox

.row { display: flex; flex-direction: row; gap: 12px; }
.row.center { justify-content: center; align-items: center; }
.grow { flex: 1; }

Grid

.grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 12px; }
.span-2 { grid-column: span 2; }

Posicionamiento

.sticky { position: sticky; top: 0; }
.abs { position: absolute; left: 10px; top: 10px; }

Responsive

@media (max-width: 600px) {
  .grid { grid-template-columns: 1fr; }
}

/* Container queries (soporte moderno) */
@container (min-width: 500px) {
  .card { display: grid; grid-template-columns: 1fr 2fr; }
}

Variables CSS (Custom properties)

:root { --space: 8px; --brand: #0d6efd; }
.btn { padding: var(--space); background: var(--brand); }

Pseudo-clases y pseudo-elementos

a:hover { text-decoration: underline; }
input:focus { outline: 2px solid #0d6efd; }
p::first-line { font-weight: 600; }

Transiciones y animaciones

.btn { transition: transform .2s ease; }
.btn:hover { transform: translateY(-2px); }
@keyframes pulse { 50% { transform: scale(1.05); } }

Arquitectura CSS (BEM)

.card {} /* bloque */
.card__title {} /* elemento */
.card--featured {} /* modificador */
  • Nombres predecibles; evita selectores demasiado profundos.
  • Componentiza estilos; limita alcance y dependencias.

Buenas prácticas

  • Prefiere layout con Flexbox/Grid sobre hacks antiguos.
  • Usa variables para temas; evita repetir valores.
  • Piensa móvil primero; optimiza fuentes/imágenes.
  • Evita !important; usa especificidad controlada.

Por qué y cuándo usar cada técnica

  • Flexbox vs Grid: usa Flexbox para alineación en una dimensión (filas/columnas); Grid para layouts en dos dimensiones (rejillas complejas). Flexbox simplifica distribución y alineación; Grid define áreas y relaciones espaciales.
  • rem vs px: rem escala con la preferencia del usuario (accesibilidad). Úsalo para tipografía y espaciamiento global; usa px cuando el pixel físico sea crítico (bordes finos).
  • Variables CSS: centralizan “tokens” de diseño (colores, espacios, tamaños) y permiten temas dinámicos sin recompilar. Útiles en diseño sistemático.
  • Media vs Container queries: media queries reaccionan al viewport; container queries al espacio real del contenedor. Prefiere container queries en componentes reutilizables.
  • box-sizing: border-box: hace que padding/border se incluyan en el tamaño total, lo que evita cálculos confusos y desbordes inesperados.

Decisiones y trade-offs

  • Resets/normalizaciones: mejoran consistencia entre navegadores, pero pueden alterar estilos nativos útiles. Aplícalos con criterio y documenta excepciones.
  • Unidades de viewport (vh): pueden fallar en móviles por barras del navegador. Considera unidades modernas (dvh, svh, lvh) cuando estén disponibles.
  • Posicionamiento absoluto: útil para overlays y badges; evita usarlo para layout principal ya que rompe el flujo y dificulta la responsividad.
  • Animaciones: añaden valor visual, pero impactan batería y performance. Respeta @media (prefers-reduced-motion) y anima propiedades que usen la GPU (transform, opacity).
  • Frameworks vs CSS nativo: frameworks aceleran el desarrollo pero añaden carga y convenciones. CSS nativo ofrece control fino; decide según contexto y equipo.

Antipatrones y errores comunes

  • Abusar de !important en lugar de mejorar la arquitectura de selectores.
  • Selectores demasiado profundos que acoplan componentes y dificultan el mantenimiento.
  • Layout con float o position cuando Flexbox/Grid resuelven mejor.
  • Hardcodear colores/espacios sin variables; complica el theming.
  • No contemplar accesibilidad del foco (:focus-visible) y estados interactivos.

Ejercicios

  1. Maqueta una tarjeta con BEM y variables (color, espacio).
  2. Crea un layout responsive con Grid (3 columnas → 1 en móvil).
  3. Haz un header sticky y un menú flex con centrado vertical.
  4. Implementa un tema claro/oscuro usando :root y variables.
